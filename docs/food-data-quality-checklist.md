# Food Data Quality Checklist

Before adding new foods to the curated database, verify the following:

- [ ] **ID Uniqueness**: Does the `id` perfectly match the standard naming convention (e.g., `category-foodname`) and is it completely unique?
- [ ] **Slug Uniqueness**: Is the `slug` URL-friendly and unique?
- [ ] **Core Nutrients**: Are `calories`, `protein`, `carbs`, and `fat` provided as valid positive numbers?
- [ ] **Serving Sizes**: Does the food have at least one valid serving size with `grams`, `unit`, and `quantity`? (Food-specific units like 'cup' or 'tbsp' must accurately represent that specific food's weight).
- [ ] **Default Serving**: Is one serving size marked as `isDefault: true`?
- [ ] **Aliases**: Have common alternative names been added to the `aliases` array for search readiness?
- [ ] **Category Validity**: Does the assigned `category` exist in `categorySchema.ts`?
- [ ] **Source Labels**: Are `source` and `sourceLabel` clearly defined?
- [ ] **Estimated Flags**: If values are estimates, is `isEstimated: true` properly set?
- [ ] **No Fake USDA Claims**: USDA is only claimed if directly derived from the official API.
- [ ] **No LLM Truth**: No final nutrition values were magically generated by LLM assumptions—they must be based on known curated sources.
